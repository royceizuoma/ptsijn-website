Content may be reorganized, presented with improved layouts, made responsive, and visually redesigned, but the original text, media references, post titles, campaign titles, dates, and embedded media should be preserved in the migration data.

## Migration Phases

### Phase 1: Freeze and Preserve

1. Keep the original WordPress XML export unchanged.
2. Keep a full copy of `/wp-content/uploads/` from hosting or FTP.
3. Run the included extraction script to create structured migration JSON and raw HTML files.
4. Store private donation data separately from public website content.

### Phase 2: Content Modeling

Create the new website content model before building pages:

- Site settings
- Pages
- Blog posts
- Outreach reports
- Galleries
- Videos
- Campaigns
- Leadership profiles
- Events/meetings
- Ministries/programs
- Forms
- Media assets
- Redirects
- Private donation archive

### Phase 3: Automated Import

Use `outputs/migration-data` as the source for the new website:

- Use JSON files for structured import.
- Use `raw-html` files when exact original page/post body content must be rendered or manually transformed.
- Use `media.json` and `media-references.json` to download/copy image assets.
- Use `redirects.csv` to configure old-to-new redirects.

### Phase 4: Manual Review

Review plugin-specific and sensitive content:

- Donation records
- Contact forms
- WooCommerce products
- Legacy theme templates
- Draft legal pages
- Placeholder/demo content
- Campaign placeholder text
- Menus with blank labels

### Phase 5: Build Modern Presentation

After content is preserved:

1. Build modern page layouts.
2. Apply new design system, typography, lighting effects, and animation.
3. Keep old content text intact.
4. Replace old WordPress shortcodes/plugin wrappers with modern components only when they do not change content meaning.
5. Verify every old public URL has a redirect.

### Phase 6: Quality Assurance

Before launch:

- Compare old and new content item counts.
- Check every image reference.
- Check every YouTube embed.
- Validate mobile layouts.
- Validate donation and contact forms.
- Validate redirects.
- Confirm private donor data is not exposed.
- Confirm legal/privacy pages are published if required.
